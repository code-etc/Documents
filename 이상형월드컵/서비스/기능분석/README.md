## Flowchart Diagram

### 로그인

> - OAuth2 로 진행된다.
> - OAuth2 은 기본적인 정보활용 동의를 구한다.
> - IDP(IDentity Provider)의 제공정보로 회원정보로 활용한다.

```mermaid
flowchart LR
    a1([시작])
    a2[소셜 로그인]
    a3{기존 정보확인}
    a4[회원정보 등록]
    a5([완료])

    a1 --> a2
    a2 --> a3
    a3 -->|없음| a4
    a3 -->|있음| a5
    a4 --> a5
```

### 게임 등록

> - __월드컵/대신정해주기__ 는 게임정보(주제 등)와 후보자를 등록한다.
> - 후보자는 이름/태그/이미지를 갖게 되며, __2개 이상의 후보자__ 등록이 요구된다.

```mermaid
flowchart LR
    a1([시작])
    a2[주제 등록]
    a31[이름 등록]
    a32[태그 등록]
    a33[이미지 등록]
    a4{추가 등록}
    a5{후보자 수 > 2}
    a6([완료])

    a1 --> a2
    a2 --> a3
    subgraph a3[후보자 등록]
        direction LR
        a31 --> a32
        a32 --> a33
    end
    a3 --> a4
    a4 -->|Yes| a3
    a4 -->|No| a5
    a5 -->|Yes| a6
    a5 -->|No| a3
```
